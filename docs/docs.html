Introduction

The Courier API enables creating courier shipments, downloading labels, and managing courier services. The API is based on the REST standard and uses JSON format for communication.

Base URL: https://developers.baselinker.com/recruitment/api
Status: The API is fully functional and ready for production use.

Authentication

All API requests require authentication using an API key. The API key must be passed in every request in the Apikey field.

API Endpoints

The API supports the following endpoints:

POST /api
All requests are sent to the same endpoint with different commands in the Command field.

GetShipmentLabel

Downloads a shipment label in the specified format.

Request parameters:

Field Type Required Description
Apikey string Yes API key
Command string Yes Must be "GetShipmentLabel"
Shipment object Yes Data for shipment identification

Shipment object structure:

Field Type Required Description
TrackingNumber string Yes* Tracking number (required if ShipperReference not provided)
ShipperReference string Yes* Sender reference (required if TrackingNumber not provided)
LabelFormat string No Label format (PDF, PNG, ZPL300, ZPL200, ZPL, EPL)

Request example:

{
    "Apikey": "ylToImatPpw6cNayhgQj",

Supported label formats

Format Description Usage
PDF Label in PDF format Default format, best quality
PNG Label in PNG format Raster image, good quality
ZPL300 ZPL code for 300 DPI printers High resolution thermal printers
ZPL200 ZPL code for 200 DPI printers Standard resolution thermal printers
ZPL ZPL code (default 300 DPI) Standard ZPL format
EPL EPL code for 203 DPI printers Epson printers

Error codes

List of possible error codes and their meanings:

ErrorLevel Description Example
0 Success - operation completed successfully Shipment created
1 Validation error - invalid data Missing required field
2 System error - technical problem Database connection error

Common validation errors:

Error Cause Solution
Missing required field Not all required fields provided Check documentation and add missing fields
Invalid format Data doesn't meet format requirements Check data format (e.g. ISO 2 country code)
Limit exceeded Field exceeds maximum length Shorten text to service limit
Unsupported service Provided service is not supported Use one of the supported services
Invalid country Country not supported by service Check supported countries for selected service
Invalid weight Weight exceeds service limit Check maximum weight for service
Invalid HS code HS code has incorrect length Use HS code with 6-10 characters
Invalid phone number Phone number format is incorrect Use only digits, no spaces or special characters
